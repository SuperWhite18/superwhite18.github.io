梳理 LLM 应用开发的三种核心架构模式，分析各自的适用场景和工程挑战。结合生产环境中的真实案例，给出架构选型建议。

## 三种架构模式

### RAG（检索增强生成）

RAG 通过将外部知识库与 LLM 结合，解决了模型"知识截止"和"幻觉"两大痛点。典型架构：

- 文档切片 &rarr; 向量化存储 &rarr; 相似度检索 &rarr; 上下文注入 &rarr; LLM 生成

适用场景：企业知识库问答、文档分析、客服系统。

### Agent（智能体）

Agent 让 LLM 拥有"自主决策"能力——它能根据任务动态规划步骤、调用工具、评估结果并修正错误。

### Tool-use（工具调用）

最轻量的模式。LLM 识别用户意图后，调用外部 API 或函数来完成特定操作。这是目前生产环境中最常见的模式。

## 架构选型建议

- **Tool-use**：简单任务，确定性高
- **RAG**：需要外部知识，但交互流程固定
- **Agent**：复杂多步骤任务，需要自主决策
